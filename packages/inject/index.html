💻 F12 / 书签工具使用 #

方式一：F12 控制台直接使用

1. 打开任意支持的 AI 网站（如 DeepSeek）

2. 按 F12 打开开发者工具

3. 在 Console 中粘贴以下代码：

Console - 一键注入 
// 加载并注入 DeepSeek 支持
const script = document.createElement('script');
script.src = 'https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/standalone.js';
script.onload = () => {
  const injector = window.AIClashInject.createInjector({ provider: 'deepseek' });
  injector.inject().then(() => {
    console.log('✅ AI Clash 已注入！使用以下命令：');
    console.log('  - window.__AI_CLASH.chat.fill(\"你好\") - 填充内容');
    console.log('  - window.__AI_CLASH.chat.send() - 发送消息');
    console.log('  - window.__AI_CLASH.chat.newChat() - 新对话');
  });
};
document.head.appendChild(script);

方式二：创建书签工具

新建一个浏览器书签，将以下内容粘贴为 URL：

Bookmarklet 
javascript:(function(){var s=document.createElement('script');s.src='https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/standalone.js';s.onload=function(){var i=window.AIClashInject.createInjector({provider:'deepseek'});i.inject().then(function(){console.log('✅ AI Clash 已注入')})};document.head.appendChild(s)})();

创建后，在任意 AI 网站点击这个书签即可自动注入。

🔌 Chrome 扩展集成 #

将 AI Clash Inject 集成到你自己的 Chrome 扩展中，通过 Extension Message API 进行通信。

1. Content Script 中注入

content.js 
import { createInjector } from '@ai-clash/inject';

// 创建使用 extension 适配器的注入器
const injector = createInjector({
  provider: 'deepseek',
  adapter: 'extension'
});

// 注入到页面
await injector.inject();

// 监听来自 background 的消息chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  if (message.type === 'EXECUTE') {
    injector.call(message.capability, message.method, ...message.args)
      .then(result => sendResponse({ success: true, result }))
      .catch(err => sendResponse({ success: false, error: err.message }));
    return true; // 保持消息通道
  }
});

2. Background 发送指令

background.js 
// 向 DeepSeek 页面发送消息
const tabs = await chrome.tabs.query({ url: 'https://chat.deepseek.com/*' });
if (tabs[0]) {
  const response = await chrome.tabs.sendMessage(tabs[0].id, {
    type: 'EXECUTE',
    capability: 'chat',
    method: 'fill',
    args: ['你好，DeepSeek!']
  });
  console.log('执行结果：', response);
}

3. Manifest 配置

manifest.json 
{
  "name": "My AI Extension",
  "manifest_version": 3,
  "permissions": [
    "tabs",
    "scripting"
  ],
  "host_permissions": [
    "https://chat.deepseek.com/*",
    "https://www.doubao.com/*",
    "https://tongyi.aliyun.com/*"
  ],
  "content_scripts": [{
    "matches": ["https://chat.deepseek.com/*"],
    "js": ["content.js"],
    "run_at": "document_start"
  }],
  "background": {
    "service_worker": "background.js"
  }
}

🤖 Puppeteer / Playwright 自动化 #

在自动化脚本中直接使用 AI Clash Inject 控制 AI 页面，适合批量处理、自动化测试等场景。

Puppeteer 示例

puppeteer-example.js 
import puppeteer from 'puppeteer';

const browser = await puppeteer.launch({ headless: false });
const page = await browser.newPage();

// 导航到 DeepSeek
await page.goto('https://chat.deepseek.com');

// 注入 AI Clash
await page.addScriptTag({
  url: 'https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/standalone.js'
});

// 执行注入并调用
await page.evaluate(async () => {
  const injector = window.AIClashInject.createInjector({
    provider: 'deepseek'
  });

  await injector.inject();

  // 填充并发送消息
  await injector.call('chat', 'fill', '如何用 Python 写一个爬虫？');
  await injector.call('chat', 'send');

  return '消息已发送!';
});

console.log('✅ 自动化完成');

Playwright 示例

playwright-example.js 
import { chromium } from 'playwright';

const browser = await chromium.launch({ headless: false });
const page = await browser.newPage();

await page.goto('https://chat.deepseek.com');

// 注入脚本
await page.addScriptTag({
  url: 'https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/standalone.js'

// 执行自动化
const result = await page.evaluate(async () => {

  await injector.inject();

console.log(result);

  • 建议使用 headless: false 方便调试
  • 可以在 evaluate 中添加事件监听来接收流式输出

📡 WebSocket 远程控制 #

通过 WebSocket 连接，实现远程对浏览器中 AI 页面的控制。适合构建集中式 AI 管理平台。

浏览器端（AI 页面）

client.html 
<script src="https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/standalone.js"></script>
<script>
  // 创建 WebSocket 连接
  const ws = new WebSocket('ws://localhost:8080');

  // 注入 AI Clash
  const injector = window.AIClashInject.createInjector({
    provider: 'deepseek',
    adapter: 'ws',
    wsUrl: 'ws://localhost:8080'
  });

  await injector.inject();

  console.log('✅ 已连接到 WebSocket 服务器');
</script>

服务器端（Node.js）

server.js 
import { WebSocketServer } from 'ws';

const wss = new WebSocketServer({ port: 8080 });

wss.on('connection', (ws) => {
  console.log('🔌 新连接建立');

  // 发送指令到浏览器
  ws.send(JSON.stringify({
    type: 'call',
    capability: 'chat',
    method: 'fill',
    args: ['你好，这是来自服务器的消息!']
  }));

  ws.send(JSON.stringify({
    type: 'call',
    capability: 'chat',
    method: 'send',
    args: []
  }));

  // 接收浏览器返回的数据
  ws.on('message', (data) => {
    const message = JSON.parse(data);
    console.log('📥 收到：', message);
  });
});

console.log('🚀 WebSocket 服务器运行在 ws://localhost:8080');

消息格式

字段 说明 示例值
type 消息类型 "call", "event"
capability 能力名称 "chat", "thinking"
method 方法名 "fill", "send", "sync"
args 参数数组 ["你好"]

📢 BroadcastChannel 跨页通信 #

使用 BroadcastChannel 在多个标签页之间传递消息，无需中心化服务器。

控制器页面

controller.html 
<script type="module">
  import { createInjector } from 'https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/index.esm.js';

  // 创建 BroadcastChannel
  const channel = new BroadcastChannel('ai-clash-channel');

  // 发送指令到 AI 页面
  function sendCommand(capability, method, ...args) {
    channel.postMessage({
      type: 'call',
      capability,
      method,
      args
    });
  }

  // 接收 AI 页面的响应
  channel.onmessage = (event) => {
    console.log('收到响应：', event.data);
  };

  // 示例：发送消息
  sendCommand('chat', 'fill', '你好!');
  sendCommand('chat', 'send');
</script>

AI 页面（被控制）

ai-page.html 
<script type="module">
  import { createInjector } from 'https://cdn.jsdelivr.net/npm/@ai-clash/inject/dist/index.esm.js';

  const injector = createInjector({
    provider: 'deepseek',
    adapter: 'broadcast',
    channelName: 'ai-clash-channel'
  });

  await injector.inject();
  console.log('✅ 已准备好接收 BroadcastChannel 消息');
</script>
ℹ️
使用场景：
  • 一个控制台同时控制多个 AI 页面
  • 去中心化的多 AI 协作系统
  • 浏览器扩展的简化通信方案

📚 API 文档 #

createInjector(options)

创建注入器实例。

参数 类型 必填 说明
provider string ✅ AI 提供者 ID：'deepseek', 'doubao', 'qianwen', 'longcat', 'yuanbao'
adapter string ❌ 适配器类型：'window' (默认), 'extension', 'ws', 'broadcast'
wsUrl string ❌ WebSocket 地址，仅 adapter='ws' 时需要
channelName string ❌ BroadcastChannel 名称，仅 adapter='broadcast' 时需要

注入器方法

inject()

注入能力到页面。

await injector.inject();

eject()

移除注入，清理页面。

injector.eject();

call(capability, method, ...args)

调用能力方法。

// 填充输入框
await injector.call('chat', 'fill', '你好 AI!');

// 发送消息
await injector.call('chat', 'send');

// 开始新对话
await injector.call('chat', 'newChat');

// 开启/关闭深度思考
await injector.call('thinking', 'sync', true);
await injector.call('thinking', 'sync', false);

// 获取思考模式状态
const state = await injector.call('thinking', 'getState');

能力与方法速查

能力 方法 参数 返回值 说明
chat newChat - { success, reason? } 开始新对话
fill text: string { success, reason? } 填充输入框
send - { success, method?, reason? } 发送消息
thinking getState - { found, enabled } 获取思考模式状态
sync wanted: boolean { success, changed, reason? } 同步思考模式到指定状态
search getState - { found, enabled } 获取智能搜索状态
sync wanted: boolean { success, changed, reason? } 同步智能搜索到指定状态
model getCurrent - { found, model? } 获取当前模型
getAvailable - { found, models: ModelInfo[] } 获取可用模型列表
select modelIdOrName: string { success, reason? } 选择模型

🔧 常见问题 #

Q: 注入后没有反应？

A: 检查以下几点：

  • 确认当前页面是支持的 AI 网站（如 chat.deepseek.com）
  • 确认页面已完全加载完成
  • 打开控制台查看是否有错误信息

Q: 如何添加事件监听接收流式输出？

A: 使用 addEventListener 监听自定义事件：

window.addEventListener('ai-clash-chunk', (event) => {
  console.log('收到内容片段：', event.detail.text);
  console.log('阶段：', event.detail.stage); // 'thinking' | 'responding'
});

window.addEventListener('ai-clash-complete', (event) => {
  console.log(event.detail.provider + ' 回答完成!');
});

Q: 如何判断当前 AI 页面是否已注入？

A: 检查全局变量是否存在：

if (window.__AI_CLASH) {
  console.log('✅ AI Clash 已注入');
} else {
  console.log('❌ 尚未注入，请先执行 createInjector().inject()');
}

Q: 支持更多 AI 平台吗？

A: 目前支持 DeepSeek、豆包、通义千问、LongCat、腾讯元宝。如需添加其他 AI，可以参考源码中的 Provider 实现自行扩展。
